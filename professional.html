Memory Cache Objects


Hotaru CMS has a number of variables that it loads into memory to speed up execution of

Plugins

$h->plugins['activeFolders'][plugin_folder] = 1;
$h->plugins['activeTypes'][plugin_type] = 1;

There are no details stored in this array. It is an index array and operates like an index on a database table. Within the Hotaru code it is called as follows: if (isset($h->plugins['activeFolders']['bookmarking'])) { // do something } ; The isset call provides a very quick response, quicker than doing a look up on array_search or some other foreach loop.

Plugin Settings

$h->pluginSettings[plugin_folder][plugin_setting];

This lists the enabled plugins for the site.

The array is a key list of the plugin names with their setting values

allPluginDetails

$h->allPluginDetails['pluginData'][plugin_folder] = $plugin;
$h->allPluginDetails['pluginFolderIndexOnClass'][plugin_class] = plugin_folder;
Hooks
$h->allPluginDetails['hooks'];
